SERVER DEPLOYMENT RUNBOOK

Sellkaro — Backend Deployment Guide

Node.js + Express + Prisma + PostgreSQL on AWS EC2, with Nginx, SSL, PM2 & GitHub Actions CI/CD. Step-by-step, exactly as deployed.

🏗️ Architecture

api.awaaz.life → AWS EC2 (Ubuntu) → Nginx → Node server on port 5000
awaaz.life → Vercel (React frontend)
Database → Supabase PostgreSQL
Email → Resend  ·  Media → Cloudinary + AWS S3
CI/CD → GitHub Actions (push to main → auto-deploy to EC2)
1
Launch EC2 Instance
  • AWS Console → EC2 → Launch Instance
  • Name: olx-server · OS: Ubuntu 22.04 LTS · Type: t2.micro (Free Tier)
  • Key pair: create new (RSA, .pem) → download & keep safe
  • Security Group — allow inbound ports: 22 (SSH), 80 (HTTP), 443 (HTTPS), 5000
  • Launch → note the Public IPv4 (e.g. 65.0.72.126)
⚠️ Public IP reboot pe badalta hai. Permanent rakhne ke liye AWS me Elastic IP allocate karke instance se attach karo (attached rahe to free).
2
SSH into EC2 (from Windows)

PowerShell me — pehle .pem permission fix, phir connect:

# PEM permission fix (one time)
icacls "C:\path\to\key.pem" /inheritance:r
icacls "C:\path\to\key.pem" /grant:r "$($env:USERNAME):(R)"

# Connect
ssh -i "C:\path\to\key.pem" ubuntu@65.0.72.126
3
Install Node, PM2, Nginx
# System update
sudo apt update && sudo apt upgrade -y

# Node.js 20 via NVM
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc
nvm install 20

# PM2 (process manager) + Nginx
npm install -g pm2
sudo apt install -y nginx git

# Verify
node --version && pm2 --version && nginx -v
4
Add Swap Memory (IMPORTANT for t2.micro)

1GB RAM npm ci/build ke liye kam padti hai — process "Killed" (exit 137) ho jata hai. 2GB swap se fix:

sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# Verify — Swap: 2.0Gi dikhna chahiye
free -h

7
Prisma Generate, Migrate & Build

Order important hai — generate PEHLE, build BAAD me:

cd ~/sellkaro/apps/server

# 1. Generate Prisma client (creates src/generated/prisma)
npm run db:generate:prod

# 2. Apply DB migrations
npm run db:migrate:prod

# 3. Build (tsc + tsc-alias + copies prisma client into dist)
npm run build
⚠️ Key fix: tsc generated Prisma .js files ko dist me copy nahi karta. Iss liye build script me copy:prisma step add kiya gaya hai jo src/generated → dist/generated copy karta hai. Warna runtime pe ERR_MODULE_NOT_FOUND: dist/generated/prisma/index.js aata hai.

10
Nginx Reverse Proxy

Config seedha sites-enabled me banao (symlink jhanjhat se bachne ke liye):

sudo tee /etc/nginx/sites-enabled/awaaz > /dev/null <<'EOF'
server {
    listen 80;
    server_name api.awaaz.life;
    location / {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        client_max_body_size 50M;
    }
}
EOF

# Remove default site, test, reload
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl reload nginx

# Test (HTTP)
curl http://api.awaaz.life/api/v1/categories

Troubleshooting (jo issues aaye)

ERR_MODULE_NOT_FOUND — dist/generated/prisma/index.js

Cannot find module '.../dist/generated/prisma/index.js'

Prisma client src/generated me banta hai, tsc use dist me copy nahi karta. Build script me copy:prisma step se fix. Manual: cp -r src/generated dist/generated → pm2 restart.

tsc-alias: not found

sh: 1: tsc-alias: not found

Stale node_modules. Root se reinstall: cd ~/sellkaro && npm ci.

npm ci "Killed" (exit 137)

Process exited with status 137 — Killed

Out of Memory (t2.micro 1GB RAM). Fix: 2GB swap add karo (Step 4).

Frontend CORS blocked

NODE_ENV=prod ko code me handle kiya gaya (pehle sirf 'production' check tha). CORS_ORIGINS me frontend domain daalo → pm2 restart olx-server --update-env.

Build errors only on EC2 (not local)

tsc -b (build mode) tsc --noEmit se strict hai. CI me asli npm run build chalao taaki errors pehle pakde jaayein.
